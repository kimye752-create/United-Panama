### 0-2. ⚠ 표현 통일 (세션 3~4 확정 — 절대 준수)
- **"JSON seed 사전 수집"** = 사용자가 사전에 자료 수집·정제하여 정적 자료 형태로 DB화 (예: JSON, PDF)
- **Gemini ≠ 능동 크롤러**. Gemini는 사용자의 자료 수집·검증 **보조 도구**
- **데이터 신뢰성** = AI 생성 능력 아님. **사용자의 사전 검증**에 기반
- 이전 버전 차이점·변경 이력은 설계서 본문에 포함 금지. 핸드오프에만 기록

### 0-3. ⚠ 작업 방식
- 짧고 빠른 판정. 달강은 의사결정 빠름 — 옵션 2~3개 주면 즉시 결정
- 코드 직접 작성 금지 → Cursor에 라우팅 (Claude 크레딧 절약)
- Cursor 보고 "X건 적재"는 반드시 `SELECT COUNT` 1회 후 신뢰
- 풀 리뉴얼이 필요한 문서는 부분 수정 대신 풀 리뉴얼 선호
- 컨텍스트 80%+ 도달 시 🔔 자동 세션 전환 + 핸드오프 생성
