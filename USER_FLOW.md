## 🚧 2·3공정 연결 지점 (향후 확장)

현재 세션 7 기준 구현은 **1공정 (시장 조사 → 보고서 1)** 만 완성. 2·3공정은 "준비 중" 페이지로 처리하되, 보고서 1의 데이터가 2·3공정의 입력이 되도록 설계.

### 보고서 1 → 보고서 2 (FOB 역산) 연결
- **입력**: 블록 4-2의 가격 포지셔닝 데이터
- **처리**: Logic A (공공 입찰 역산) / Logic B (민간 소매 역산)
- **출력**: 보고서 2 (수출 전략 제안서)
- **공정 잠금**: 보고서 1 완료 후에만 "[2공정 실행]" 버튼 활성화

### 보고서 1 → 보고서 3 (파트너 매칭) 연결
- **입력**: panama_distributors 4건 + Case 판정 결과
- **처리**: AHP 기반 PSI 점수화 (엔진⑥)
- **출력**: 보고서 3 (파트너 Top 10 리스트)
- **공정 잠금**: 보고서 1·2 완료 후에만 "[3공정 실행]" 버튼 활성화

## ⚠ 절대 원칙 (플로우 협상 불가)

1. **10초 이내 응답** — 분석 파이프라인은 10초를 넘기면 안 됨 (ARCHITECTURE.md 원칙)
2. **분석 버튼 시 11개 사이트 직접 호출 금지** — Supabase 조회만 허용
3. **Phase B(web_search)는 freshness_checker 기반 조건부** — 7일 이내 데이터 있으면 스킵
4. **공정 잠금 구조** — 1 → 2 → 3 순차 실행, 중간 스킵 불가
5. **사용자는 분석 근거 데이터를 보유할 필요 없음** — 시스템이 자동 수집·구조화
6. **PDF는 @react-pdf/renderer만 사용** — puppeteer는 Vercel 서버리스에서 느림
7. **한국어 UI + 한국어 보고서** — 영문 병기는 회사명·정부기관명·INN만
